# 用途と外部仕様
- 考え方 : 練習phaseの概念。これまでの練習を練習phase1とする。movesを練習phase2とする
## 用途 : phase1 : buttons 練習
- 用途 : buttonsに慣れる用。これはこれでマストなので、削除はしない
- 仕様 : 左右 : 左右はランダム
    - 要は、これまでとほぼ同じ（ただしmovesは表示しない）
- 実装 : 既存実装を軸にする
## 用途 : phase2 : moves 練習
- 用途 : あくまでphase1を身に着けたあとに練習が必要なもの
- 仕様 : 左右 : ひとまず、全missionsを一通りやりきるごとに、左、右、をトグル
    - movesの表示 : 1行目にmoves表示をする。先頭に（右向き）（左向き）も追加表示する
    - buttons文字列 : challenge_phase（後述）で分岐し、現在の左右に応じて生成する。
    
# 詳細な実装
## toml : challenge_phase
- challenge_phase = "1_buttons"
- challenge_phase = "2_moves"
## mission生成
- challenge_phase に応じて生成する
    - challenge_phase = "1_buttons" の場合、既存実装そのまま
    - challenge_phase = "2_moves" の場合、現在の左右に応じてmissionの左右を変更する
## 1行目に（右向き）などを表示
- challenge_phase に応じて表示内容を加工する
